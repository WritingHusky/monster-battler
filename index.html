Welcome to my Monster Batttler Project!

This project is based on the popular website Pokémon Showdown, which simulate pokemon battles for users to interact with. This project is broken down into three integral parts:

  1. Monster Battling System: Implemented in Java, this portion of the project is where all the logic of the turn in a battle is handled. The system is designed to simulate a turn in a battle given the current state of the battle.

    View Front-end Code

  2. Monster Battle API: This spring-boot API serves as the backend for the project, handling the creation of battles, the execution of turns, simulating AI opponents, and more. The API is designed to be robust, efficient, and scalable, ensuring a seamless experience for users.

    View API Code

  3. React Frontend: Although aesthetics take a backseat, the React frontend (using TypeScript) provides a functional interface for users to engage with the battle system.

    View Battle System Code
