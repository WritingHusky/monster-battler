Monster Battling System

Implemented in Java, this part of the project handles the logic for simulating a turn in a battle based on the current battle state.

Monster Battle API

This Spring Boot API serves as the project's backend, managing battle creation, turn execution, AI simulation, and more. It is designed to be robust, efficient, and scalable for a seamless user experience.

React Frontend

Although aesthetics are secondary, the React frontend (using TypeScript) provides a functional interface for users to interact with the battle system.
